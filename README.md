# Discord Bot (MVP)

Бот для Discord на **NestJS** и **Necord** (Discord.js). Поэтапная разработка: сначала базовый MVP из трёх функций, затем расширение.

## MVP — три основные функции

### 1. Конструктор сообщений
- Публикация сообщений от имени бота с профессиональным оформлением
- Создание и сохранение шаблонов сообщений
- Кастомное форматирование: абзацы, отступы, иконки, изображения
- Структурированные сообщения для администраторов сервера

### 2. Роли по реакции
- Выдача ролей при нажатии кнопки или реакции (эмодзи) на сообщение
- Самостоятельный выбор ролей пользователями
- Упрощение управления доступом к каналам

### 3. Система журналов (логи)
- Отправка событий в назначенные каналы:
  - join / leave
  - message logs
  - moderation logs
  - channel logs
  - ban / kick logs
- Удобный мониторинг активности для администрации

## Стек

- **Node.js** + **NestJS**
- **Necord** — интеграция Discord.js с NestJS
- **discord.js** — работа с Discord API
- **@nestjs/config** — конфигурация (переменные окружения)

## Запуск

### 1. Установка зависимостей

```bash
npm install
```

### 2. Переменные окружения

Скопируйте пример и подставьте свой токен:

```bash
cp .env.example .env
```

В `.env` укажите:

- **`DISCORD_TOKEN`** — токен бота из [Discord Developer Portal](https://discord.com/developers/applications) (Bot → Reset Token / View Token).
- Для **дашборда** (логин через Discord, «мои серверы», отправка сообщений): **`DISCORD_CLIENT_ID`** (Application ID приложения), **`DISCORD_CLIENT_SECRET`** (OAuth2 → Client Secret), **`DISCORD_CALLBACK_URL`** (например `http://localhost:3000/api/auth/callback`), **`SESSION_SECRET`** (любая случайная строка для сессий). В Developer Portal в OAuth2 → Redirects добавь `http://localhost:3000/api/auth/callback`.
- **`DISCORD_GUILD_ID`** — только для разработки. ID тестового сервера, чтобы команды появлялись сразу. В продакшене **не задавай** эту переменную: тогда команды регистрируются глобально и бот работает на всех серверах, куда его пригласили.

### 3. Настройка бота в Discord

В Developer Portal для приложения:

1. **Bot** — включите нужные **Privileged Gateway Intents** (например, Server Members Intent, Message Content Intent, если нужны сообщения и участники).
2. **OAuth2 → URL Generator** — выберите scope `bot` и права (например, Send Messages, Manage Roles, Read Message History и т.д.) и пригласите бота на сервер.

### 4. Запуск в режиме разработки

```bash
npm run start:dev
```

В консоли должно появиться сообщение о входе бота (например, `Bot logged in as YourBot#1234`) и строка `HTTP server: http://localhost:3000`.

**Дашборд:** открой в браузере [http://localhost:3000](http://localhost:3000) — страница «Войти через Discord», после входа список серверов (где есть бот и у тебя есть права), по клику на сервер — форма «Отправить сообщение» и блок шаблонов. Подробный гайд для фронта (логин, API, шаблоны, отправка от имени бота): **[docs/FRONTEND_GUIDE.md](docs/FRONTEND_GUIDE.md)**.

**Чтобы slash-команды появились сразу:** добавь в `.env` переменную `DISCORD_GUILD_ID` — ID своего тестового сервера (ПКМ по иконке сервера → «Копировать ID»; для этого нужна включённая настройка «Режим разработчика» в Discord). Без этого новые команды могут подтягиваться до часа.

## Бот на многих серверах (продакшен / продажа)

Когда бота будут приглашать на разные сервера (покупатели, клиенты):

1. **Один инстанс бота, один токен** — не нужно поднимать отдельного бота под каждый сервер.
2. **В продакшене не указывай `DISCORD_GUILD_ID`** в `.env`. Тогда Necord регистрирует slash-команды **глобально**, и они доступны на любом сервере, куда пригласили бота.
3. **Данные уже разделены по серверам**: настройки логов, каналы и т.д. хранятся в `data/guilds.json` по `guildId`. У каждого сервера свой набор настроек, они не пересекаются.
4. Приглашение: каждый клиент добавляет бота к себе через одну и ту же ссылку (OAuth2 URL с нужными правами). После добавления на сервере появятся команды `/post`, `/reaction-role`, `/logs` — каждый админ настраивает только свой сервер.

Итого: для продажи/многосерверного режима просто **не задавай `DISCORD_GUILD_ID`** при деплое — и один бот будет обслуживать все сервера с изолированными настройками.

## Как тестировать в Discord

После запуска бота (`npm run start:dev`) зайди на сервер, куда приглашён бот, и проверь три функции так.

### 1. Конструктор сообщений

- В любом канале введи слэш и выбери команду **`/post`**.
- Укажи:
  - **канал** — куда отправить сообщение (можно текущий);
  - **заголовок** и **описание** — текст эмбеда;
  - **картинка** (необязательно) — URL картинки.
- Подтверди. В выбранном канале должно появиться оформленное сообщение от бота с заголовком, описанием и при необходимости картинкой.

### 2. Роли по реакции

- Создай на сервере роль (например «Новичок»). В настройках сервера **роль бота должна быть выше этой роли**, иначе бот не сможет её выдавать.
- Вызови **`/reaction-role`**:
  - **канал** — где отправить сообщение с кнопкой;
  - **роль** — какую роль выдавать по нажатию;
  - **текст** (необязательно) — подпись над кнопкой.
- В канале появится сообщение с кнопкой (название кнопки = название роли). Нажатие выдаёт роль, повторное нажатие — снимает.

### 3. Логи (вход/выход)

- Создай канал для логов (например `#join-leave`). Права на просмотр можно оставить только у администрации.
- Вызови **`/logs set`**:
  - **тип** — «Вход / Выход (join-leave)»;
  - **канал** — выбери созданный канал.
- Когда кто-то заходит на сервер или выходит, в этом канале будут появляться сообщения с ником, аватаром, ID и временем. Чтобы отключить: **`/logs off`** → выбери тип «Вход / Выход».

Настройки логов сохраняются в папке `data/` в файле `guilds.json`.

## Структура проекта

```
src/
├── main.ts                 # Точка входа (инициализация бота, без HTTP)
├── app.module.ts           # Корневой модуль
├── discord/                # Подключение к Discord (Necord)
│   ├── discord.module.ts
│   └── discord.update.ts   # События ready, warn, error
├── message-constructor/    # MVP #1 — конструктор сообщений
├── reaction-roles/         # MVP #2 — роли по реакции
└── logs/                   # MVP #3 — система логов
```

В модулях реализованы: slash-команды (`/post`, `/reaction-role`, `/logs set` / `/logs off`), обработка кнопки для ролей и отправка логов join/leave в выбранный канал.
