# Авторазвёртывание Discord-сервера по шаблону

Техническая возможность, ограничения Discord API и вариант реализации через дашборд.

---

## 1. Краткий ответ

**Да, реализовать можно**, в том числе с управлением из дашборда: выбор шаблона и кнопка «Установить сервер» (Install). Почти всё из списка (категории, каналы, роли, эмодзи, сообщения, автороли, логи) делается через Discord API и уже имеющийся функционал бота. Ограничения в основном по лимитам API и по объёму данных (эмодзи/стикеры — размер и количество).

---

## 2. Что можно автоматизировать через API/бота

| Элемент | Возможность | Как |
|--------|-------------|-----|
| **Категории** | ✅ Полностью | `guild.channels.create({ name, type: ChannelType.GuildCategory, position })` |
| **Текстовые/голосовые каналы** | ✅ Полностью | Создание канала с `parentId` (категория), `permission_overwrites`, topic, rateLimit |
| **Роли** | ✅ Полностью | Создание ролей, permissions, color, hoist, mentionable, порядок (position) |
| **Кастомные эмодзи** | ✅ Полностью | Загрузка по URL или Buffer; лимиты сервера (50 стандартных, 50 анимированных при бусте) |
| **Серверные стикеры** | ✅ Через API | Создание стикеров (формат, размер — см. документацию Discord) |
| **Сообщения и эмбеды** | ✅ Полностью | Отправка в нужный канал (уже есть в боте) |
| **Кнопки / автороли** | ✅ Полностью | Компоненты в сообщении + привязка «кнопка/реакция → роль» (уже реализовано) |
| **Каналы логов** | ✅ Полностью | Создание каналов + сохранение в настройках бота (уже есть) |
| **Иконка/баннер сервера** | ✅ При правах | `guild.setIcon()`, `guild.setBanner()` при наличии прав у бота |
| **Права каналов** | ✅ Полностью | `permission_overwrites` при создании канала (роли по id созданных ролей) |

**Итого:** Всё из п.6 концепции (структура, роли, эмодзи, стикеры, сообщения, автороли, логи) технически автоматизируемо. Полностью через дашборд — да: пользователь выбирает шаблон и нажимает «Установить».

---

## 3. Ограничения и нюансы

- **Лимиты Discord:** количество каналов (500), категорий (50 каналов в категории), ролей (250), эмодзи (50/50 без буста). В рамках одного «типового» сервера обычно укладываемся.
- **Порядок создания:** сначала роли (чтобы задать permission_overwrites по id ролей), потом категории, потом каналы в категориях, затем сообщения и привязки авторолей.
- **Эмодзи/стикеры:** нужны файлы или URL картинок; при «экспорте» шаблона с другого сервера — скачать и хранить у себя или отдавать URL.
- **Ссылки в шаблоне:** в шаблоне храним не «конкретный id канала с сервера-донора», а логическую привязку («канал с именем rules», «роль с именем Member»). При установке подставляем уже созданные id.

## 4. Архитектура (как сделать через дашборд)

### 4.1 Шаблон сервера (данные)

Один шаблон — структура данных (например JSON или таблицы в БД), которая описывает:

- **Мета:** id, название (например «Lineage 2 Community»), описание, превью.
- **Роли:** имя, цвет, permissions (битовая маска), hoist, mentionable, порядок.
- **Категории:** имя, порядок (position).
- **Каналы:** имя, тип (text/voice/forum и т.д.), категория (ссылка по имени или порядку), topic, permission_overwrites (по имени роли), rateLimit и т.д.
- **Эмодзи:** имя, URL картинки (или id файла в нашем хранилище).
- **Стикеры:** имя, описание, URL/файл.
- **Сообщения:** «в канал X отправить сообщение»: текст, эмбед (title, description, color, image), компоненты (кнопки с customId).
- **Привязки авторолей:** какое сообщение (по привязке к каналу + порядку или временному id в шаблоне), какая кнопка/эмодзи, какая роль (по имени роли в шаблоне).
- **Логи:** какие каналы использовать для joinLeave, messages, moderation, channel, banKick (по имени канала в шаблоне).

Такой шаблон можно: создавать вручную (админ-интерфейс), импортировать из JSON или (частично) собирать с «образцового» сервера (экспорт структуры и сообщений ботом).

### 4.2 Установка (Install) из дашборда

Типичный сценарий:

1. Пользователь заходит в дашборд, выбирает сервер (гильдию), куда ставим шаблон.
2. Выбирает шаблон из списка (например «Lineage 2 Community Server»).
3. Нажимает кнопку **«Установить сервер»** (Install Server).
4. Бэкенд:
   - проверяет, что бот на сервере и есть нужные права (Manage Roles, Manage Channels, Manage Server, Read/Send Messages и т.д.);
   - создаёт роли по шаблону, запоминает соответствие «имя роли в шаблоне → id созданной роли»;
   - создаёт категории, запоминает «имя категории → id»;
   - создаёт каналы (с parentId, permission_overwrites по созданным ролям);
   - загружает эмодзи и стикеры;
   - отправляет сообщения (с эмбедами и кнопками) в нужные каналы, запоминает «какое сообщение в шаблоне → messageId»;
   - регистрирует привязки «роль по реакции» (messageId + emoji/кнопка → roleId из созданных ролей);
   - настраивает каналы логов в нашем хранилище (как уже делается для логов).
5. В дашборде показываем результат: «Установлено» или список ошибок (например «нет прав на создание роли»).

Всё это можно вызывать одним действием из дашборда (одна кнопка «Установить»), а на бэке — одна «длинная» операция (или очередь задач при очень больших шаблонах).

### 4.3 Дашборд

- **Страница «Шаблоны серверов»:** список доступных шаблонов (название, описание, превью). Для каждого — кнопка «Установить» (если выбран текущий сервер).
- **На странице сервера:** блок «Установить шаблон» — выбор шаблона из выпадающего списка + кнопка «Установить сервер». После установки — обновить список каналов/ролей или показать сообщение об успехе.
- Опционально (для админов): «Создать шаблон из текущего сервера» — экспорт структуры в наш формат (категории, каналы, роли, сообщения из каналов, настройки логов и авторолей). Это отдельный большой этап; минимальный MVP — ручное описание шаблонов в БД/JSON и только кнопка «Установить».

## 5. Связка с Discord Server Templates (опционально)

- Можно использовать **Discord Server Template** как первый шаг: пользователь создаёт сервер из шаблона Discord (категории, каналы, роли уже есть), приглашает бота, а в дашборде нажимает **«Дозаполнить по нашему шаблону»**. Тогда бот только: создаёт недостающие роли/каналы (если нужно), заливает эмодзи/стикеры, отправляет сообщения, настраивает автороли и логи.
- Либо полностью свой путь: бот ставится на **пустой** сервер пользователя и по нашему шаблону создаёт всё с нуля (категории, каналы, роли, сообщения, автороли, логи). Оба варианта реализуемы; «всё с нуля» даёт полный контроль и не зависит от лимитов Discord-шаблонов.

## 6. Что делать по шагам (реализация)

1. **Модель данных шаблона** — сущности в БД (или один JSON-документ): шаблон, роли, категории, каналы, эмодзи, «сообщения к отправке», привязки авторолей, настройки логов.
2. **Сервис установки** — один метод типа `installTemplate(guildId, templateId)`: создаёт роли → категории → каналы → эмодзи/стикеры → сообщения → привязки авторолей → настройки логов. Использовать существующие сервисы (каналы, роли, сообщения, логи, реакция-роли).
3. **API для дашборда** — например: `GET /api/templates` (список шаблонов), `POST /api/guilds/:id/install-template` с телом `{ templateId }`. Ответ — успех или ошибки.
4. **UI в дашборде** — список шаблонов и кнопка «Установить сервер» / «Установить шаблон» на выбранный сервер.
5. **Экспорт шаблона** (позже) — с «образцового» сервера читать структуру и сообщения и сохранять в нашу модель шаблона.

## 7. Итог

- **Возможно ли реализовать авторазвёртывание сервера:** да.
- **Можно ли это делать через дашборд:** да — выбор шаблона и одна кнопка «Установить сервер».
- **Что автоматизируется полностью:** структура (категории, каналы), роли, эмодзи, стикеры, сообщения и эмбеды, кнопки авторолей, настройка логов. Ручная часть по сути только: пригласить бота на сервер и выдать ему нужные права; остальное — один клик в дашборде.
- **Технические ограничения** — лимиты Discord и необходимость хранить шаблон в понятном формате (роли/каналы по «логическим» именам, чтобы при установке подставлять реальные id).

Если нужно, следующий шаг — описать конкретную схему БД для шаблонов и сигнатуры API под ваш текущий бэкенд (NestJS + дашборд).
